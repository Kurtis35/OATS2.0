Update the Rockhaven Wedding Guest Transport Portal to implement a fully functional wedding transport booking system.

The portal must use Google Sheets as the data store while keeping the website 100% frontend-only so it works with Git deployment and hosting on Netlify.

Use only:

HTML
CSS
Vanilla JavaScript

Do NOT use:

Backend servers
Databases
Frameworks

The portal must be fast, mobile-first, and extremely simple to use.

IMPORTANT CONTEXT

The Wedding Portal is a section of an existing website, not a full website.

Users access it by clicking “Wedding Portal” on the main website.

Inside this section there are three internal portal pages.

These pages must live inside a folder:

/wedding-portal/

Structure:

/wedding-portal
   index.html
   portal.html
   services.html
   style.css
   script.js

This allows the portal to load like:

site.com/wedding-portal
site.com/wedding-portal/portal
site.com/wedding-portal/services

This ensures it works perfectly when deployed on Netlify.

The design must feel minimal, luxury, and world-class.

The overall feeling should be calm, premium, and wedding-focused.

GOOGLE SHEETS INTEGRATION

All booking data must be sent to Google Sheets via Google Apps Script Web App.

Use this endpoint:

https://script.google.com/macros/s/AKfycbyKgq_pUO2wgYmsZxLDWE81v3MIaLg4exyGh_x7CNY/dev

Form submissions must send data using a POST request.

Example logic:

fetch(SCRIPT_URL, {
  method: "POST",
  body: JSON.stringify(data)
})

Admins will be able to:

• View bookings in Google Sheets
• Export guest lists
• Manage passenger counts

No backend server should exist.

SHUTTLE SCHEDULE DATA

Pickup schedules must load dynamically from this published Google Sheets CSV:

JavaScript should:

Fetch the CSV

Admins must be able to change pickup times directly in Google Sheets and the website should automatically update.
